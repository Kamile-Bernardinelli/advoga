# Story 1.1 — Ingestion Pipeline

> **ID:** 1.1 · **Epic:** EP-1 Combat Core
> **Status:** Draft
> **Agente executor:** Dara (@data-engineer) + ultracode (tagging em lote)
> **DoD Drop 1 que cobre:** "dados reais" — sem este pipeline não há questões para nenhuma outra story de EP-1

---

## User Story

**Como** Kamile,
**quero** que o sistema tenha as ~10 edições mais recentes da OAB 1ª fase carregadas com as respostas corretas e a tag de matéria/subtema,
**para que** eu possa começar a fazer provas reais com diagnóstico imediato.

---

## Contexto Técnico

Fontes públicas (sem login necessário):
- Provas + gabaritos: `examedeordem.oab.org.br` e `oab.fgv.br/arq/...` (PDF, URLs com padrão previsível).
- Gabarito definitivo FGV (não o preliminar).
- Tagging matéria + subtema: LLM-assistido por lote. Micro-tópico = Drop 2.
- Validade Drop 1: recentes presumidas vigentes. Anuladas = `validade_status = 'anulada'` conforme gabarito definitivo.
- Ordem: recente→antigo (D-02 do brief).

Schema destino: `exames`, `questoes` (RF-01 do PRD).

---

## Acceptance Criteria

- [ ] AC-1.1.1 (RF-01): ≥ 10 edições OAB 1ª fase mais recentes ingeridas no Supabase da Kamile. Campo `exames.edicao` populado com número romano.
- [ ] AC-1.1.2 (RF-01): Cada questão tem: `enunciado`, `alt_A..D`, `gabarito` (gabarito definitivo FGV), `materia_id`, `subtema_id`, `fonte_url`, `validade_status`.
- [ ] AC-1.1.3 (RF-12): Questões anuladas têm `validade_status = 'anulada'` e `validade_motivo` preenchido (ex.: "Anulada pelo gabarito definitivo FGV — sem resposta correta").
- [ ] AC-1.1.4 (RF-01): Script de ingestão reproduzível (não depende de SQL manual). Documentado em `docs/guides/ingestion.md`.
- [ ] AC-1.1.5 (RF-01): Tagging matéria/subtema com cobertura ≥ 95% das questões ingeridas. Questões sem tag registradas em log de auditoria.
- [ ] AC-1.1.6: Seed de `materias` e `subtemas` cobrindo as 20 disciplinas da 1ª fase com pesos de incidência corretos (grupos A/B/C conforme PRD §5).
- [ ] AC-1.1.7: Nenhum dado sensível (Supabase keys, gabaritos não-públicos) commitado ao repositório.
